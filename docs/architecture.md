# Архитектура рекламной платформы

## Обзор

Рекламная платформа представляет собой систему на основе микросервисов, предназначенную для управления рекламными кампаниями и объявлениями. Платформа состоит из нескольких взаимосвязанных сервисов, которые обрабатывают различные аспекты рабочего процесса рекламы, включая управление кампаниями, показ объявлений, аутентификацию пользователей, биллинг, аналитику и администрирование.

## Компоненты системы

### 1. Сервис Ads API

Сервис Ads API построен на Go и фреймворке Gin. Он обеспечивает основную функциональность создания и управления рекламными кампаниями и объявлениями.

**Основные возможности:**
- RESTful API для управления кампаниями и объявлениями
- База данных MySQL для постоянного хранения данных
- Tarantool для кэширования
- RabbitMQ для очереди сообщений
- Метрики Prometheus для мониторинга
- Логирование запросов и обработка таймаутов

**Конечные точки:**
- `GET /health` - Проверка работоспособности
- `POST /campaigns` - Создать новую кампанию
- `GET /campaigns/{id}` - Получить кампанию по ID
- `PUT /campaigns/{id}` - Обновить кампанию
- `DELETE /campaigns/{id}` - Удалить кампанию
- `POST /ads` - Создать новое объявление
- `GET /ads/{id}` - Получить объявление по ID
- `PUT /ads/{id}` - Обновить объявление
- `DELETE /ads/{id}` - Удалить объявление
- `GET /metrics` - Метрики Prometheus

**Модели данных:**
- **Campaign**: Содержит user_id, name и status
- **Ad**: Содержит campaign_id, title и status

### 2. Сервис Auth API

Сервис Auth API построен на Python и FastAPI. Он обеспечивает аутентификацию пользователей, авторизацию и операции биллинга.

**Основные возможности:**
- Аутентификация и управление сессиями пользователей
- Биллинг и обработка платежей
- Управление балансом
- История транзакций
- Интеграция с базой данных MySQL

**Конечные точки биллинга:**
- `POST /billing/deposit` - Пополнить счет пользователя
- `POST /billing/withdraw` - Снять средства со счета пользователя
- `GET /billing/balance/{user_id}` - Получить баланс пользователя
- `GET /billing/transactions/{user_id}` - Получить историю транзакций
- `POST /billing/charge-for-ad` - Списать за создание объявления
- `POST /billing/charge-for-creative` - Списать за создание креатива

### 3. Сервис Analytics API

Сервис Analytics API предоставляет возможности аналитики данных и отчетности для рекламной платформы.

**Основные возможности:**
- Агрегация и анализ данных
- Расчеты показателя CTR (click-through rate)
- Метрики эффективности
- Инструменты отчетности
- Интеграция с базой данных ClickHouse

### 4. Панель администратора

Панель администратора - это веб-приложение на Django, которое предоставляет административные возможности для платформы.

**Основные возможности:**
- Управление пользователями
- Надзор за кампаниями
- Финансовая отчетность
- Мониторинг системы
- Управление конфигурацией

## Инфраструктура

### Базы данных
- **MySQL**: Основная реляционная база данных для хранения структурированных данных, таких как кампании, объявления, пользователи и балансы
- **ClickHouse**: Колоночная база данных для аналитики и отчетности
- **Tarantool**: In-memory база данных для кэширования и операций в реальном времени

### Очереди сообщений
- **RabbitMQ**: Используется для асинхронной обработки задач, таких как показ объявлений, уведомления и сбор аналитики

### Мониторинг
- **Prometheus**: Сбор и мониторинг метрик
- **Grafana**: Визуализация дашбордов
- **AlertManager**: Система оповещений

### Развертывание
- **Docker**: Контейнеризация сервисов
- **Docker Compose**: Оркестрация для локальной разработки
- **Ansible**: Автоматизация подготовки инфраструктуры и развертывания

## Безопасность

- Аутентификация на основе JWT
- Контроль доступа на основе ролей
- Валидация и очистка входных данных
- Безопасные протоколы связи
- Ограничение частоты запросов для предотвращения злоупотреблений

## Масштабируемость

- Архитектура микросервисов позволяет независимое масштабирование
- Уровень кэширования снижает нагрузку на базу данных
- Асинхронная обработка для тяжелых операций
- Возможности горизонтального масштабирования

## Поток данных

1. Пользователи проходят аутентификацию через Auth API
2. Пользователи создают кампании и объявления через Ads API
3. Операции биллинга обрабатываются через Auth API
4. Показ объявлений и отслеживание кликов происходят асинхронно через RabbitMQ
5. Данные аналитики собираются и сохраняются в ClickHouse
6. Отчеты и дашборды генерируются из собранных данных
